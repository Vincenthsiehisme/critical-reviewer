# 05 — 麥肯錫框架套用指南

## 使用說明

這個檔案是框架的操作細則。
顧問發言前如需確認框架的正確套用方式，讀取對應段落。
每個框架包含：核心邏輯、套用步驟、常見誤用。

---

## MECE
**Mutually Exclusive, Collectively Exhaustive**
不重疊、不遺漏

**套用步驟**：
1. 識別產出中的分類或選項清單
2. 測試「不重疊」：任兩個項目之間，有沒有本質上相同的部分？
3. 測試「不遺漏」：在這個分類維度上，有沒有重要的類別沒有出現？
4. 說明具體發現，而非只說「違反 MECE」

**常見誤用**：
- 說「這份文件不夠 MECE」但沒說哪裡重疊、哪裡遺漏 ← 不合格
- 用 MECE 挑細節用詞，而非結構問題 ← 不合格

---

## Issue Tree
**問題樹狀拆解**

**套用步驟**：
1. 定義根問題（一句話）
2. 展開第一層：這個問題的主要成因或面向有哪些？（2–4 個）
3. 展開第二層：每個成因可以再拆成哪些子問題？
4. 標記產出目前在哪個節點上作業
5. 判斷這個節點是否是根因，或只是症狀

**風險顧問的失敗路徑變體**：
根問題 = 「這個方案為什麼會失敗」，往下拆前提條件與執行環節。

**常見誤用**：
- 只畫一層就停止，沒有到可行動的深度 ← 不合格
- 把 Issue Tree 當成清單列，沒有層次關係 ← 不合格

---

## So What / Why So
**洞察雙向測試**

**套用步驟**：
1. 找出產出中的每個洞察或結論句
2. 往下問 So What：「這意味著什麼具體行動或設計決策？」
   - 如果無法回答：這個洞察是裝飾，沒有實際作用
3. 往上問 Why So：「這個洞察的根據是什麼？」
   - 如果無法回答：這個洞察是假設，不是事實

**常見誤用**：
- 只問其中一個方向 ← 必須雙向
- 對所有洞察一律套用，不挑重要的 ← 聚焦在 2–3 個關鍵洞察即可

---

## 假設驅動
**先假設再驗證**

**套用步驟**：
1. 掃描產出，列出所有「用戶希望⋯⋯」「我們認為⋯⋯」「應該會⋯⋯」類型的句子
2. 對每個假設判斷類型：
   - ✅ 已驗證：有訪談、數據或直接觀察支撐
   - ⚠️ 未驗證：合理推測，但沒有依據
   - ❌ 不可驗證：無法用現有方法確認
3. 對未驗證的假設按「驗證成本 × 假設錯誤的後果」排優先序

**常見誤用**：
- 只說「這份文件有很多假設」但不列出 ← 不合格
- 把已驗證的事實也列進去 ← 會稀釋真正的風險假設

## Prioritization Matrix
**影響力 × 可行性**

**套用步驟**：
1. 列出產出中的行動選項或方案
2. 在兩個維度上評估每個選項：
   - 影響力：對目標的貢獻程度（高 / 中 / 低）
   - 可行性：在已知限制下能否執行（高 / 中 / 低）
3. 找出「高影響力但被低估或忽略」的選項
4. 指出機會成本：做這個，不做什麼？

**常見誤用**：
- 只評估被選中的選項，不評估被放棄的選項 ← 缺少機會成本視角
- 沒有說明評估依據，只給結論 ← 不合格

---

## Pyramid Principle
**結論先行，論點支撐**

**套用步驟**：
1. 找出產出的結論句（如果埋在最後，標記為問題）
2. 檢查論點層：有沒有 2–3 個清晰的支撐論點？
3. 檢查數據 / 觀察層：每個論點有沒有具體依據？
4. 整合段使用時：先寫核心主張，再寫論點，最後寫行動建議

**常見誤用**：
- 把 Pyramid Principle 用來挑文字順序，而非邏輯結構 ← 不合格
- 整合段寫成三個顧問意見的摘要列點，沒有統一主張 ← 不合格
