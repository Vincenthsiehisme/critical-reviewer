# Examples

這個目錄存放完整的審查輸出範例，供 Claude 在 Phase 4 需要對照格式時讀取。

## 使用時機

只有在 `06-output-templates.md` 的格式說明不夠清楚時才讀取，正常審查流程不需要載入。

## 範例清單

| 檔案 | 情境 | 召集顧問 | 特點 |
|------|------|---------|------|
| `full-example-travel-product.md` | 旅遊產品分類系統概念驗證 | 使用者研究員 ＋ 風險顧問 | 包含搜尋發現、獨立收斂、無顧問張力 |

## 範例結構說明

每個範例展示完整的四階段流程：
1. **背景確認**：萃取自 session 的事實背景
2. **快速掃描**：框架缺口診斷與顧問召集決定
3. **顧問獨立調查**：各顧問的搜尋發現 ＋ 框架分析（顧問間互相隔離）
4. **交叉比對整合**：獨立收斂、單一發現、顧問張力的三層結構

## 新增範例的規範

新範例必須包含：
- `**搜尋發現**` 欄位（即使無差距也要標示「此欄省略」）
- `（獨立調查）` 標示在顧問名稱後
- 整合段使用 `🔴 獨立收斂 / 🟡 單一發現 / ⚡ 張力` 三層結構
